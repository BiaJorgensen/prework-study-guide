Prework Study Guide

HTML

  • HTML describes the structure of a webpage.
  • The head element contains information about the webpage.
  • The title element is the only user visible element in the head element. It names the tab of a webpage.
  • The body element represents the visible content shown to the user.
  • Elements nested in other elements are called Child Elements.
  • Child elements are nested in Parents Elements.
  • Element is everything in between opening and closing tags.
  • Tag is only what is in between the angle brackets.

CSS

  • CSS is used to make webpage more user friendly, by adding colors, aligning text, and other styles.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External CSS file sheet is preferred because it keeps rules separate from HTML file.
  • Inline CSS file styles an element directly on the HTML file and should only be used for quick and permanent changes.
  • Internal Style Sheet adds CSS rules directly in HTML head element and applies only to that particular page.
  • It is best practice to separate code into different files based on use, this is called Separation of Concerns.
  • Avoid repeating rules, when possible, combine 2 or more selectors (using comma) when creating a rule. DRY - Do Not Repeat Yourself!
  • Create Class Attributes as selectors (i.e.: .card) to add styles to certain elements, not all.

Git

  • Never edit directly from the main branch
  • Git command must proceed every Git statement.
  • git status: checks what branch we are currently on.
  • git checkout branch name: moves working branch to another branch.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Always use 'git pull original main' (change "main" to the branch name you are branching from) to pull most recent version of branch. This way you will not be working from an outdated code version.
